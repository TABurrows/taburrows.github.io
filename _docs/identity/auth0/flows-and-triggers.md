Processes which can be extended with Actions are called Flows.

Each Flow is made up of one or more Triggers and represents the Logical Pipeline through which Information moves during a Single Point in the Auth0 Journey.

Multiple Actions can be added to a Trigger, with each Action executing in the order in which it was placed.

Some Triggers exec synchronously, blocking the flow in which they are involved, and some are executed asynchronously.
