Alerting Policies


Recommended best practice for alerts:

- Use multiple notification channels so you avoid a single point of failure.
- Configure alerting on symptoms and not necessarily causes.
- Customize your alerts to the audience need.



Valid targets for Cloud Monitoring uptime alert notifications.

- 3rd party service
- webhook
- Pub/sub
- email
- SMS



Uptime checks -> Supported Resource Types:
- Instance
- App Engine
- Elastic Load Balancer
- Cloud Run Service
- URl
- Internal IP
- K8s LoadBalancer Service
