Database - Business Case


Make the Business Case: the pros, cons, cost/benefits or risks/rewards 

Perform risk and cost benefit analysis on a Cloud Migration project estimate the cost associated with DB migration


A business case is used to justify why the company should take on a project

Goals:
- Improve efficiency
- Drive innovation
- Reduce costs
- Create value
- Increase productivity

How:
- Lift and shift
- Improve and move
- Rebuild

Risks/rewards:
- Cost/benefit
- Risk/reward
- Time
- Budget

Start with where you are, define where you want to be ... (sometimes improve and move is a better strategy)

If you first build a PoC then migrate some simple applications


Strategies (depends on the appliction):
- Rewrite as a Cloud-native appliction
- Lift and shift, then optimize
- Improve on-premises, then move to the cloud

The Best Candidate application for moving to the Cloud first:
- Few dependencies
- Not mission-critical
- No compliance issues
