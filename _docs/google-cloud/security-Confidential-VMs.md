Security - Confidential VMs

Confidential Computing: encrypting data-in-use = encrypting traffic as it crosses the bus from the CPU to Memory and back; encrypting each VM Guest individually with a unique Key-pair (Secure Encrypted Virtualization = SEV)

- Runs on AMD EPYC processors/machine types
- AMD Developed Secure Encrypted Virtualization (SEV)

- Can only be enabled when you CREATE a new VM
- Only requires the ticking of checkbox in the UI, or 1-2 lines of additional parameters on the CLI
- Confidential VMs provide END-TO-END ENCRYPTION:
  - ENCRYPTION-AT-REST
  - ENCRYPTION-IN-TRANSIT
  - ENCRYPTION-IN-USE

Confidential Computing VMs provide:
Isolation:
- Encryption keys are generated by the AMD Secure Processor (SP) during VM Creation and reside solely within the AMD System-on-Chip (SOC)
  ( nb. these keys are not accessible to Google )
Attestation ( to attest means: "to declare something exists or is the case or to certify formally" ):
- with Confidential VMs this is achieved using Virtual Trusted Platform Module (vTPM) Attestation
