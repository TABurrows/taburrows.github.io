Microservices

- Service Boundaries match Business Boundaries
- Codebase is modular
- Each service can be implemented independently updated, deployed and scaled


Journey to the Cloud (tm) is not an all or nothing: you can re-architect and migrate legacy applications in an incremental manner


Implement stateless components for scalability

Cloud Functions can both fire/trigger on the reception of data, or be called upon to Process, Transform and Store data

Application Components should start up quickly to enable efficient scaling, and shut down gracefully when they receive a termination signal

Cache Application Data that is frequently accessed or that is Computationally intensive to calculate

Use Cloud Endpoints to DEVELOPE, DEPLOY, PROTECT and MONITOR APIs based on the OpenAPI Spec or with gRPC

You can implement an API as a FACADE or ADAPTER LAYER to legacy applications that cannot be refactored or moved to the Cloud
